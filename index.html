📑 Obfuscation Feature Overview

🔹 Default Configuration

Overview: Enables comment removal and compression by default, along with 3 to 6 randomized obfuscation techniques such as control flow restructuring, callback wrapping, staged execution injection, delayed branching, RC4 string encoding, global variable renaming, anti-debugging, self-defense protection, and object key obfuscation.

Effectiveness: Suitable for rapid protection of most web content.

🟡 Note: Default obfuscation does not hide the DOM structure—it only wraps the source code. For enhanced protection, combine with additional feature options.

🔹 Text Fragmentation & Camouflage

Overview: Splits all static text inside the <body> into multiple custom nodes <obfusx>, preventing real text from appearing directly in the DOM.

Effectiveness: Prevents content copying and scraping by turning meaningful text into noise, enhancing text-level protection.

🔹 Random Attribute Injection

Overview: Injects randomly generated attribute names and pseudo-class names into DOM elements to create structural noise and increase difficulty in targeting styles and scripts.

Effectiveness: Disrupts selector targeting and style analysis, increasing obfuscation complexity.

🔹 Tag Semantics Mutation

Overview: Replaces standard tags (e.g., <div>) with non-semantic tags (e.g., <obfusx-com>).

Effectiveness: Breaks semantic structure and interferes with automated recognition and style matching.

🔹 ID Replacement & Mutation

Overview: Replaces all static IDs with hashed or randomized strings.

Effectiveness: Prevents quick identification of key elements via ID selectors, strengthening structural protection.

🔹 Invisible Signature

Overview: Injects invisible custom signature data into the DOM, carrying a unique identifier that can be verified via code.

Effectiveness: Enables traceability and ownership verification for original content.

How to Query: Copy and run the following code in your browser console:

getComputedStyle(document.documentElement).getPropertyValue("--obfusx-signature")

// Your signature will appear here

🔹 Link Camouflage

Overview: Modifies <a> tag attributes such as href, rel, and target, or nests invalid child elements to disrupt semantic recognition and crawler parsing.

Effectiveness: Prevents links from being directly identified, scraped, or mirrored.

🔹 Fake Elements

Overview: Inserts non-functional DOM elements near key nodes, assigning misleading IDs or class names to simulate real structure.

Effectiveness: Increases node count and disrupts structural analysis and debugging tools.

🔹 Anchor Disruption

Overview: Injects empty anchors and fake IDs to interfere with browser navigation and crawler jump logic.

Effectiveness: Scrambles page positioning logic and disrupts automated anchor recognition.

🔹 Modularization

Overview: Splits script logic into multiple function modules, shuffling execution order and naming conventions.

Effectiveness: Enhances code irreversibility and increases analysis difficulty.

🔹 Character Disruption

Overview: Inserts zero-width characters (such as \u200B, \u200C) into text nodes to create semantic noise and interfere with parsing and pattern matching.

Effectiveness: Obstructs copying, pasting, and selector targeting—enhancing content extraction resistance and anti-scraping capability.

🟡 Note: This feature will be automatically disabled when “Text Fragmentation & Camouflage” is enabled to avoid conflicts.

How to Query: Copy and run the following code in your browser console:

const node = document.querySelector('body'); // 或其他包含静态文本的元素 如 <html>
if (node) {
  const codes = [...node.textContent].map(c => c.charCodeAt(0));
  console.log("字符码位:", codes);
  console.log("字符个数:", codes.length);

  const hasGhost = codes.filter(c => c === 8291).length;
  console.log("幽灵字符数量:", hasGhost);
}

// Inserted characters will appear here

🔹 Shortcut Key Blocking

Overview: Listens for keyboard events and prevents default actions to block common shortcut keys.

Effectiveness: Effectively stops non-technical users from quickly accessing developer tools or cloning your content.

🔹 Anti-Embedding Protection

Overview: Detects window.top !== window.self and enforces redirection or rendering block to prevent iframe embedding.

Effectiveness: Prevents hotlinking, nested invocation, and malicious framing.

🔹 Developer Tools Disruption

Overview: ObfusX detects tampering of native objects like RegExp, Date, and Function via their toString() methods, monitors performance.now() for debugger suspension, and checks window dimensions (e.g., outerWidth - innerWidth) to identify devtools usage.

Effectiveness: In most browser environments, this can interrupt debugging, clear page content, or trigger forced redirection. Even if the page remains open, only obfuscated code will be visible.

🔹 Download Protection

Overview: Uses CSP headers or dynamic loading to prevent external .css / .js files from being directly saved.

Effectiveness: Disables “Save As” functionality for static assets, protecting your external resources.

🔹 Access Rate Limiting

Overview: Tracks visit timestamps and frequency, triggering blocking logic when thresholds are exceeded.

Effectiveness: Prevents high-frequency scraping and malicious refresh behavior.

🔹 Domain Lock & Redirection

Overview: Checks if the current domain is whitelisted; unauthorized domains trigger redirection or rendering block.

Effectiveness: Prevents unauthorized mirroring, hotlinking, and cross-domain access.

🔹 Watermark Protection

Overview: Inserts watermark text via CSS background layers or pseudo-elements.

Effectiveness: Reinforces content ownership and enhances visual anti-copy protection.

❓ Frequently Asked Questions (FAQ)

Does ObfusX store my source code?

No. ObfusX is a purely front-end tool. All processing is done locally in your browser—your source code is never uploaded or stored.

Is my code completely secure after obfuscation?

No. Any JavaScript that runs in the browser must be parsed and executed—there’s no such thing as “absolute security.” ObfusX significantly increases reverse engineering difficulty and improves protection.

Why is the obfuscated code larger than the original?

ObfusX applies multiple layers of protection. The increase in code size is expected and reflects the added complexity for defense.

Does obfuscation affect SEO?

Yes. Modules like Text Fragmentation, Tag Mutation, Link Camouflage, Character Disruption, and Anchor Interference may impact crawler parsing. Choose based on your SEO needs.

Can ObfusX obfuscate my standalone JS files?

No. ObfusX focuses on full-page HTML obfuscation. It does not process standalone JS files. For scripts, use tools like javascript-obfuscator first, then apply ObfusX to the complete page.

Can I restore my code after obfuscation?

No. ObfusX obfuscation is irreversible. Always back up your original files before applying protection.
